# 生产发布手册

**最后更新：** 2026-03-20

## 1. 发布前确认

发布前先确认四件事：

1. 构建期 `NEXT_PUBLIC_*` 已锁定，且和目标域名一致。
2. 运行时 `Sanity`、邮件服务、Webhook Secret 已准备好。
3. `deploy/cce/kustomization.yaml` 将指向本次镜像 tag。
4. 当前仍按单副本发布，不在本次顺手做多副本扩容。

## 2. 推荐发布顺序

### 2.1 准备构建期变量

```bash
cp .env.build.example .env.build.production
```

至少检查：

- `SWR_REGION`
- `SWR_ORG`
- `IMAGE_NAME`
- `PUSH_IMAGE=true`
- 全部需要公开注入的 `NEXT_PUBLIC_*`

### 2.2 构建并推送镜像

```bash
./build.sh 0.1.0 production
```

### 2.3 准备运行时变量

```bash
cp deploy/cce/secret.example.yaml deploy/cce/secret.yaml
```

分别补值：

- `deploy/cce/configmap.yaml`
- `deploy/cce/secret.yaml`

### 2.4 更新集群清单

至少确认：

- `deploy/cce/kustomization.yaml` 中的镜像地址和 tag
- `deploy/cce/ingress.yaml` 中的 Host/TLS Secret
- `deploy/cce/serviceaccount.yaml` 是否满足当前集群镜像拉取策略

### 2.5 发布到 CCE

```bash
kubectl apply -k deploy/cce
```

## 3. 发布后检查

- `kubectl rollout status deployment/isbim-web -n isbim-web`
- 打开首页、`/newsroom`、`/case-studies`、`/contact`
- 检查 `/api/revalidate`
- 检查 `/studio`
- 提交联络表单验证 Sanity + 邮件链路
- 检查 CDN 视频、图片、Open Graph 资源是否正常

## 4. 回滚入口

### 4.1 镜像回滚

- 把 `deploy/cce/kustomization.yaml` 的 tag 改回上一个稳定版本。
- 重新执行 `kubectl apply -k deploy/cce`。

### 4.2 配置回滚

- 如果问题来自 `ConfigMap` / `Secret`，回滚对应文件内容后重新 apply。
- 如果问题来自 `NEXT_PUBLIC_*` 变化，不能只回滚运行时变量，必须回滚镜像。

## 5. 发布时最容易混淆的地方

- `NEXT_PUBLIC_*` 属于构建期优先变量，改了就要重建镜像。
- `SANITY_API_TOKEN`、`RESEND_API_KEY` 这类密钥只属于运行时，不要放到 `.env.build.production`。
- 和 Jarvis `web` 不同，isBIM 的正式发布一定要把 `Sanity webhook` 和邮件 API 一起纳入验收。
