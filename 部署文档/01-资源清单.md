# 资源清单

**最后更新：** 2026-03-20

## 1. 文档目的

这份文档用于回答三个问题：

- `isbim-official-web` 正式部署需要哪些资源。
- 当前仓库里已经有哪些资源定义可以直接复用。
- 还有哪些资源仍然缺失，或者还没有形成稳定事实来源。

配套阅读：

- [03-华为云CCE部署手册.md](./03-华为云CCE部署手册.md)
- [04-生产发布手册.md](./04-生产发布手册.md)
- [05-环境变量参考.md](./05-环境变量参考.md)

## 2. 应用与构建产物清单

| 类别 | 资源 | 当前值或模式 | 备注 |
|---|---|---|---|
| 部署单元 | `isbim-web` | Next.js SSR | 单一前端服务，不是静态站点 |
| 镜像名 | `isbim-official-web` | 可通过 `IMAGE_NAME` 覆盖 | `build.sh` 默认值 |
| 镜像标签规则 | `<version>-<mode>` | 例如 `0.1.0-prod`、`0.1.0-local` | `build.sh` 统一生成 |
| 镜像仓库 | SWR | `SWR_REGION` + `SWR_ORG` | 例如 `<swr-region>/<swr-org>/isbim-official-web:<tag>` |
| 构建入口 | `./build.sh` | 推荐统一入口 | 复用当前 `Dockerfile`，不直接改应用代码 |
| 容器入口 | `node server.js` | Next.js standalone | 来自 `Dockerfile` 的 runner 阶段 |

## 3. 域名与入口资源清单

| 资源 | 当前值或模式 | 作用 | 当前来源 |
|---|---|---|---|
| 香港站公开域名 | `https://www.isbim.com.hk` | 站点主入口 | `deploy/cce/ingress.yaml`、`deploy/cce/configmap.yaml` |
| Sanity Studio 入口 | `https://<site>/studio` | 同站内 Studio | 由同一个 Next.js 容器提供 |
| Sanity API / CDN | `*.sanity.io` | 动态内容读取、Webhook 来源 | 外部托管依赖 |
| 视频 CDN | `NEXT_PUBLIC_VIDEO_CDN_URL` | 通用视频资源 | `.env.build.example`、`deploy/cce/configmap.yaml` |
| 产品特性视频 CDN | `NEXT_PUBLIC_FEATURE_VIDEO_CDN_URL` | 产品页 feature 视频 | `.env.build.example`、`deploy/cce/configmap.yaml` |
| 通用媒体域名 | `NEXT_PUBLIC_MEDIA_URL` | 图片/图标/字体 CDN 回退 | `.env.build.example`、`deploy/cce/configmap.yaml` |
| CCE 命名空间 | `isbim-web` | Kubernetes 逻辑隔离 | `deploy/cce/namespace.yaml` |

说明：

- 当前仓库里明确落地的 Host 仍是 `www.isbim.com.hk`。

## 4. 数据、存储与外部服务清单

| 资源类型 | 资源 | 是否必需 | 当前状态 | 说明 |
|---|---|---|---|---|
| Headless CMS | Sanity | 必需 | 已接入 | 外部托管，不在本仓库自建容器 |
| Sanity 写入 Token | `SANITY_API_TOKEN` | 强烈建议 | 已有模板 | 联络表单写入 Sanity 与部分服务端写操作依赖 |
| Sanity Webhook Secret | `SANITY_WEBHOOK_SECRET` | 强烈建议 | 已有模板 | 保护 `/api/revalidate` |
| 邮件服务 | Resend | 当前默认必需 | 已有模板 | `EMAIL_PROVIDER=resend` 时使用 |
| 邮件备份服务 | Brevo | 可选但建议 | 已有模板 | 作为 Resend 的回退提供商 |
| 邮件收件人/发件人 | `CONTACT_EMAIL_TO`、`EMAIL_FROM_*` | 必需 | 已有模板 | 与联络表单通知链路绑定 |
| 视频/媒体存储 | CDN / OBS / 公网对象存储 | 按当前内容基本必需 | 已有变量模板 | 视频资源当前优先按 CDN 方式投放 |
| 分布式限流存储 | Redis / 华为云 DCS | 多副本时必需 | 预留变量 | 当前单副本可不启用 |

## 5. CCE / Kubernetes 资源清单

`deploy/cce/` 当前已经具备最小可用清单：

- `namespace.yaml`
- `serviceaccount.yaml`
- `configmap.yaml`
- `secret.example.yaml`
- `deployment.yaml`
- `service.yaml`
- `ingress.yaml`
- `kustomization.yaml`
- `README.md`

说明：

- `secret.yaml` 不会直接提交到仓库，必须从 `secret.example.yaml` 复制生成。
- 这套清单只覆盖 `isbim-web` 一个工作负载，不会派生出第二个 CMS Deployment。

## 6. 端口、服务与流量入口

| 资源 | 当前值 | 说明 |
|---|---|---|
| 容器端口 | `3000` | `Dockerfile` 与 `deployment.yaml` 一致 |
| Service 形态 | `ClusterIP` | `80 -> 3000` |
| Ingress | `www.isbim.com.hk` | 示例 TLS Secret 为 `isbim-web-tls` |
| 探针路径 | `GET /api/revalidate` | 当前用作临时 startup/readiness/liveness probe |

## 7. 当前可确认的资源规格

`deploy/cce/deployment.yaml` 中当前基线是：

- 副本数：`1`
- Requests：`500m CPU / 1Gi RAM`
- Limits：`2 CPU / 2Gi RAM`

如果走华为云生产环境，可按下面方式理解：

- Pod 资源是 Kubernetes 的调度基线，不等于节点购买规格。
- 节点池建议至少能稳定承载一个 `Node 20 + Next.js SSR` 工作负载，并预留日志、探针和发布缓冲。
- 如果后续接入 Redis / DCS 做多副本扩容，需要重新评估节点和出口带宽。

补充说明：

- `NEXT_PUBLIC_*` 变量虽然按“构建期”分类，但运行时也建议在 `ConfigMap` 中保持同名值，避免服务端读取与前端产物口径分叉。
- 和 Jarvis `web` 不同，这里没有 `STRAPI_URL`、`STRAPI_API_TOKEN` 之类的 Strapi 变量。

## 9. 当前明确存在的资源缺口

当前优先级最高的缺口是：

1. 真实 CI/CD 仍未补齐，当前只有 `build.sh` 和 `deploy/cce/*` 作为手工/流水线共用基线。
2. 内地加速域名、备案、CDN/WAF 回源链路还没有形成仓库内的固定实值。
3. 多副本所需的 `Redis / DCS` 限流改造还未完成，所以运行策略仍应保持单副本。
4. `Sanity`、`Resend`、`Brevo`、媒体 CDN 的生产凭据和出口连通性还需要在真实环境做最终验收。
