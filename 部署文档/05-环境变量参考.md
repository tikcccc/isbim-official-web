# 环境变量参考

**最后更新：** 2026-03-20

## 1. 文档目的

这份文档专门回答三件事：

- `isbim-official-web` 现在应该保留哪些 env 模板。
- 本地开发、本地 Docker、华为云 CCE 三种场景分别该用哪些文件。
- 哪些变量属于构建期，哪些属于运行时。

## 2. 先看结论

当前 env 结构已经统一成“模板入库、实值本地化”，并显式区分“构建期”和“运行时”：

| 用途 | 仓库保留的模板 | 本机或部署时生成的实值副本 |
|---|---|---|
| 本地开发 | `.env.local.example` | `.env.local` |
| 本地直接生产模式 | `.env.production.example` | `.env.production` |
| 镜像构建 | `.env.build.example` | `.env.build.local`、`.env.build.production`、`.env.build` |
| 本地 Docker 运行时 | `.env.runtime.example` | `.env.runtime` |
| 华为云 CCE 运行时 | `deploy/cce/configmap.yaml`、`deploy/cce/secret.example.yaml` | `deploy/cce/secret.yaml` |

使用原则：

1. `*.example` 才是仓库里的标准模板。
2. `.env.local`、`.env.production`、`.env.build.*`、`.env.runtime` 都是本机或 CI/CD 工作副本，不作为仓库事实来源。
3. `NEXT_PUBLIC_*` 必须在镜像构建前确定，因为它们会进入前端产物和 `next.config.ts`。
4. 运行时密钥只放在 `.env.runtime` 或 `deploy/cce/secret.yaml`，不要混进构建期文件。
5. 当前仓库没有 `cms` 子应用，也没有 Strapi 专属 env 命名空间。

## 3. 场景速查表

| 场景 | 用哪些文件 | 至少要填什么 | 不该用什么 |
|---|---|---|---|
| 本地开发 | `.env.local.example -> .env.local` | `NEXT_PUBLIC_SANITY_*`、站点 URL、联络表单相关运行时变量 | `deploy/cce/*`、`.env.runtime` |
| 本地直接生产模式 | `.env.production.example -> .env.production` | 与生产接近的 `NEXT_PUBLIC_*`、Sanity Token、Webhook Secret、邮件变量 | `deploy/cce/*`、`.env.build.*` |
| 本地 Docker | `.env.build.example -> .env.build.local`，`.env.runtime.example -> .env.runtime` | 构建期 `SWR/NEXT_PUBLIC_*`，运行时 `Sanity/邮件/Webhook` | `.env.local`、`.env.production`、`deploy/cce/*` |
| 华为云 CCE | CI/CD 注入与 `.env.build.production` 同名的构建变量；运行时用 `deploy/cce/configmap.yaml` + `secret.yaml` | 正式域名、Sanity、邮件、Webhook、媒体 CDN | `.env.runtime`、`.env.local`、`.env.production` |

## 4. 文件分层说明

### 4.1 `.env.local.example -> .env.local`

适用场景：

- `npm run dev`

至少要填：

- `NEXT_PUBLIC_SITE_URL`
- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET`
- `NEXT_PUBLIC_SANITY_API_VERSION`
- `SANITY_API_TOKEN`
- `SANITY_WEBHOOK_SECRET`
- `EMAIL_PROVIDER`
- `CONTACT_EMAIL_TO`
- `EMAIL_FROM_INTERNAL`
- `EMAIL_FROM_USER`
- `RESEND_API_KEY`

按需填写：

- `NEXT_PUBLIC_MEDIA_URL`
- `NEXT_PUBLIC_VIDEO_CDN_URL`
- `NEXT_PUBLIC_FEATURE_VIDEO_CDN_URL`
- `BREVO_API_KEY`
- `RATE_LIMIT_REDIS_URL`
- `NEXT_CACHE_DIR`

开发环境建议：

- `EMAIL_PROVIDER` 默认用 `resend`
- `EMAIL_FROM_INTERNAL` / `EMAIL_FROM_USER` 用 `@resend.dev`，避免开发期就依赖正式域名校验

### 4.2 `.env.production.example -> .env.production`

适用场景：

- 本机执行 `npm run build && npm run start`
- 本机验证接近生产的服务端行为

至少要填：

- 全部 `NEXT_PUBLIC_*`
- `SANITY_API_TOKEN`
- `SANITY_WEBHOOK_SECRET`
- `EMAIL_PROVIDER`
- `CONTACT_EMAIL_TO`
- `EMAIL_FROM_INTERNAL`
- `EMAIL_FROM_USER`
- 与当前邮件提供商匹配的 API key

说明：

- 这份文件主要用于“直接跑进程”的生产模拟，不是 CCE 的唯一事实来源。
- 如果目标是容器化发布，优先使用 `build.sh + deploy/cce/*` 这套分层。

### 4.3 `.env.build.example -> .env.build.local / .env.build.production`

适用场景：

- `./build.sh <version> local`
- `./build.sh <version> production`

至少要填：

- `SWR_REGION`
- `SWR_ORG`
- `NEXT_PUBLIC_SITE_URL`
- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET`
- `NEXT_PUBLIC_SANITY_API_VERSION`

通常还会填：

- `IMAGE_NAME`
- `DOCKER_PLATFORM`
- `PUSH_IMAGE`
- `NEXT_PUBLIC_MEDIA_URL`
- `NEXT_PUBLIC_VIDEO_CDN_URL`
- `NEXT_PUBLIC_FEATURE_VIDEO_CDN_URL`

注意：

- 不要把 `SANITY_API_TOKEN`、`RESEND_API_KEY`、`BREVO_API_KEY` 这类运行时密钥写进这里。
- 这份文件会被 `build.sh` 当作 Docker BuildKit 的 `next_env` secret 输入，主要目的是让 `next build` 拿到稳定的公开变量。

### 4.4 `.env.runtime.example -> .env.runtime`

适用场景：

- 本地 Docker：`docker run --env-file .env.runtime`

至少要填：

- `NODE_ENV`
- 全部运行时需要的 `SANITY_*`
- `EMAIL_PROVIDER`
- `CONTACT_EMAIL_TO`
- `EMAIL_FROM_INTERNAL`
- `EMAIL_FROM_USER`
- 与当前邮件提供商匹配的 API key

建议同步保留：

- 全部 `NEXT_PUBLIC_*`

原因：

- 虽然浏览器端公开变量已经在构建期写入产物，但服务端代码仍可能在运行时读取同名值。
- 把 `ConfigMap` / `.env.runtime` 中的公开变量和构建期保持一致，最不容易出错。

### 4.5 `deploy/cce/configmap.yaml` + `deploy/cce/secret.yaml`

适用场景：

- 华为云 CCE 正式运行

`configmap.yaml` 里应放的非敏感值：

- `NODE_ENV`
- `NEXT_PUBLIC_SITE_URL`
- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET`
- `NEXT_PUBLIC_SANITY_API_VERSION`
- `NEXT_PUBLIC_MEDIA_URL`
- `NEXT_PUBLIC_VIDEO_CDN_URL`
- `NEXT_PUBLIC_FEATURE_VIDEO_CDN_URL`
- `EMAIL_PROVIDER`
- `CONTACT_EMAIL_TO`
- `EMAIL_FROM_INTERNAL`
- `EMAIL_FROM_USER`

`secret.yaml` 里应放的敏感值：

- `SANITY_API_TOKEN`
- `SANITY_WEBHOOK_SECRET`
- `RESEND_API_KEY`
- `BREVO_API_KEY`
- `RATE_LIMIT_REDIS_URL`

补充说明：

- 当前 `RATE_LIMIT_REDIS_URL` 只是为未来多副本限流预留，不代表代码已经切过去。
- 如果后续启用 `NEXT_CACHE_DIR` + 共享存储，也应该同步进 CCE 运行时环境。

## 5. 变量分类速查

### 5.1 构建期优先

- `SWR_REGION`
- `SWR_ORG`
- `IMAGE_NAME`
- `DOCKER_PLATFORM`
- `PUSH_IMAGE`
- `NEXT_PUBLIC_SITE_URL`
- `NEXT_PUBLIC_SANITY_PROJECT_ID`
- `NEXT_PUBLIC_SANITY_DATASET`
- `NEXT_PUBLIC_SANITY_API_VERSION`
- `NEXT_PUBLIC_MEDIA_URL`
- `NEXT_PUBLIC_VIDEO_CDN_URL`
- `NEXT_PUBLIC_FEATURE_VIDEO_CDN_URL`

### 5.2 运行时优先

- `NODE_ENV`
- `SANITY_API_TOKEN`
- `SANITY_WEBHOOK_SECRET`
- `EMAIL_PROVIDER`
- `CONTACT_EMAIL_TO`
- `EMAIL_FROM_INTERNAL`
- `EMAIL_FROM_USER`
- `RESEND_API_KEY`
- `BREVO_API_KEY`
- `RATE_LIMIT_REDIS_URL`
- `NEXT_CACHE_DIR`

## 6. 当前和 Jarvis `web` 的差异

- 这里没有 `STRAPI_URL`、`STRAPI_API_TOKEN`、`STRAPI_WEBHOOK_SECRET`。
- 这里的内容系统变量是 `Sanity`，不是 Strapi。
- 这里的运行时变量还要覆盖邮件发送与联络表单链路。
