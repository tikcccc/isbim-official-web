1.頁面切換轉場動畫：
傾斜轉場動畫解決方案 (概念)

這套方案旨在重現 Palantir 風格的、包含傾斜面板和 Logo 的三階段頁面轉場動畫。

1. 核心技術

Framer Motion: 用於處理動畫狀態（進場/退場）和時間序列。

Tailwind CSS: 用於實現傾斜面板的視覺樣式。

2. 階段 1：全局設定 (根佈局 Layout)

使用 Framer Motion 的 <AnimatePresence> 元件包裹 Next.js 的頁面 children。

設定 <AnimatePresence> 為 mode="wait"，確保舊頁面動畫完全結束後，新頁面才開始載入。

AnimatePresence 會監聽路由路徑 (pathname) 的變化，以此作為觸發動畫的 key。

3. 階段 2：轉場元件 (Overlay)

建立一個獨立的、全螢幕的 PageTransitionOverlay 元件，它將被 AnimatePresence 控制。

此元件包含兩層：

背景層 (傾斜面板)：一個使用 Tailwind CSS (skew-x) 製作的深色傾斜面板。

前景層 (Logo)：您的 Logo。Logo 本身會被反向傾斜，以抵銷背景的傾斜，使其保持視覺上的垂直。

4. 階段 3：動畫序列 (Animation Timeline)

這是一個精心編排的三階段動畫：

退場 (Exit)：

觸發換頁時，傾斜面板從左側滑入 (scaleX) 覆蓋畫面。

面板完全覆蓋後，Logo 淡入 (opacity) 顯示在中央。

等待 (Wait)：

AnimatePresence 在幕後（動畫遮罩下）安全地卸載舊頁面、載入新頁面。

進場 (Enter)：

新頁面載入完成後，Logo 首先淡出 (opacity) 消失。

Logo 消失後，傾斜面板從右側滑出 (scaleX)，優雅地揭露出新頁面的內容


Palantir 網站頁面轉場動畫的核心 CSS。

我們來看看您找到的這段 CSS 程式碼 (位於 eea07c1dad90f887.css 檔案的末尾)：

CSS

.ptcom-design__pageContainer__ec3r6t {
    /* 這裡定義了動畫會用到的變數 */
    --gray-wipe-x: -110%; /* 灰色面板的 X 軸位置 */
    --rotate: 28deg;      /* 這就是您看到的「傾斜角度」！*/
    
    /* 這裡是容器本身的樣式 */
    bottom: 0;
    left: 0;
    overflow: hidden; /* 隱藏超出範圍的動畫面板 */
    pointer-events: none; /* 讓它可以被點擊穿透 */
    position: fixed;  /* 覆蓋整個螢幕 */
    right: 0;
    top: 0;
    z-index: 925;     /* 確保它在頁面內容之上 */
}

/* 這是傾斜面板 (灰色) */
.ptcom-design__pageShapeGray__ec3r6t {
    background-color: #767676; /* 灰色 */
    top: -50vh;
    /* 關鍵：使用上面定義的變數來執行動畫 */
    transform: translate(var(--gray-wipe-x)) rotate(var(--rotate)); 
    z-index: 925;
}

/* 這是 Logo */
.ptcom-design__palantirLogo__ec3r6t {
    left: 50%;
    opacity: 0; /* 預設隱藏 */
    position: absolute;
    top: 50%;
    transform: translate(-50%, -50%);
    z-index: 950; /* 在灰色面板之上 */
}
這段 CSS 完美印證了我們在 transition_solution_concept.md（您右側的文件）中制定的三階段方案：

pageContainer (固定容器)：就是您找到的 .ptcom-design__pageContainer__ec3r6t。

pageShapeGray (傾斜面板)：.ptcom-design__pageShapeGray__ec3r6t，它使用 rotate(var(--rotate)) 來實現傾斜。

palantirLogo (Logo)：.ptcom-design__palantirLogo__ec3r6t，它被 z-index: 950 放在最上層，並預設 opacity: 0。

JavaScript (GSAP 或 Framer Motion) 會在換頁時動態修改 --gray-wipe-x、opacity 等值，來執行我們之前討論的完整三階段動畫。
